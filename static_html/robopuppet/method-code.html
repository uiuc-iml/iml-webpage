Software Architecture

A ROBOPuppet software system consists of three components: 1) Arduino code for sending encoder values to the serial port of the host computer, 2) a Python script for reading from the serial port, processing it, and sending joint commands to the robot's controller and 3) the robot's safety filter which produces safe motion at the motors by combining joint commands with the robot's dynamic model and model of the environment (if available).

The example code here must be adapted to your own system

Supported Systems

Windows and Linux systems are supported. Mac support may be possible with some effort.

Installation Instructions

Step 0. Install Python 2.7 and the Arduino software. Follow the Klamp't installation tutorial for your system to install the Klamp't Python API (on Windows, the binaries are fine).

Step 1. Clone the RoboPuppet git repository from Github.

Step 2. Set up your Arduino to run the script in RoboPuppet/Arduino/read_5_pins.ino. This file should be edited to reflect your Arduino's configuration.

Step 3. Edit the RoboPuppet/settings.py file to reflect your system's configuration. In particular, make note of the serial port that your Arduino is connected to.

Step 4. Test your connection by running the arduinoSimControl.py Python script from the RoboPuppet/ folder.

Step 5. To do safety-filtered control you must compile some C++ binaries. On Windows you may download the binaries. On Linux you must use the CMake system to compile the binaries, as described in RoboPuppet/README.md. Follow the instructions in the readme to do so (you will need to set the KLAMPT_ROOT environment variable when you call cmake).
