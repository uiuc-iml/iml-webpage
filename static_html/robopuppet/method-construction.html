METHOD - Construction

Download the CAD model of the target robot

Many robots have CAD files available for download. Any file format that can be opened in FreeCAD can be used. Additional file formats may require a different software choice. For the TX90L robotic arm, we downloaded the *.stp files.

Inspect each joint surface to be articulated - use these observations to determine the size of the puppet

The joint axes of most robotic joints are centered on the joint surface. This was the case for all but one joint in the TX90L robotic arm. These joints are perfect candidates for the Joint Encoder Assemblies and Pocket Geometries. In order to use the pocket geometries, simply center the encoder void(the portion of the pocket geometry that contains the encoder's base) on the joint surface. A size for the final puppet can be determined by looking at the smallest joint that is to contain a standard joint encoder assembly and scaling that joint surface to fit the pocket geometries. The robot should be scalled around this value.

Joints that do not have the joint surface to fit the generic joint fixture will require the fixture for that joint to be modified. In our puppet this includes the joint between link 4 and link 5 and the joint between link 5 and link 6. In order to accommodate the potentiometer in the smaller joint surface area on link 4 for the joint between link 4 and link 5 the generic joint fixture was move deeper into the link and an access tunnel was added to allow access. The joint between link 5 and link 6 is very different from other joints in the robot, for this joint a custom pocket geometry was made that was just large enough to accommodate the potentiometer base in the side of the link.

We found that the puppet for the TX90L should be scaled to 30% of the target robot's size.

The joint surface between joint 5 and joint 6 was too small for a standard joint encoder assembly. A custom pocket geometry, shown in pink, was created and used instead.

 

The joint surface between link 4 and link 5 was too small to support the encoder assembly. However the link was deep and had space further away from the joint surface that was suitable fore the encoder assembly. The pocket geometry was combined with an access tunnel in the size of a large cylinder, the encoder assembly was moved further into the link (as can be seen here in the green bracket).

 

 

Create a watertight mesh from the CAD files, scaled to the appropriate size

Many CAD files are not watertight, this makes them unsuitable for printing. We used FreeCAD to joint all link surfaces and create a water tight mesh of each articulated link.

Open the CAD file in FreeCAD and make it watertight.

Select a link, or if the link is in several distinct pieces several parts, from the list in the 'Labels and Attributes' menu.(Multiple pieces can be selected by pressing the ctrl key while clicking on the label. On the left you can see that link 2, the lower shoulder link, is made of 11 pieces. We have selected all of those pieces to be combined into one.

With the pieces selected click on the Join button. This is circled in red on the left. This joins all the pieces into one and converts any open wires into solid faces.

NOTE - Even if the link is already in one piece, you must complete this step in order to close any open faces and render the link watertight so that it will print properly.

NOTE - If the join button is not visible go to VIEW > WORKBENCH > ALL

 

 

 

 

 

Create and export the mesh

Once the link has been joined, select it and go to MESHES > CREATE MESH FROM GEOMETRY. Accept the default mesh tolerance of 0.10.

Select the newly created mesh in the Labels and Attributes menu(this is typically named "mesh##" where ## is the order of the meshes you have created). Go to MESHES > EXPORT MESH.

Save the mesh as an *.stl file to the desired location with an appropriate name.

 

 

 

 

 

Import the mesh into TinkerCAD and resize it

Log into TinkerCAD

Create a new file and name it appropriately. (Files can be renamed by going to the DESIGN > PROPERTIES menu.)

Go to the Import menu on the left hand side of the screen, select FILE and click on 'CHOOSE FILE'

Navigate to the mesh created in the last step and click 'OPEN'

This will populate the file name and allow you to chose a size for your imported file. Enter the amount you would like to scale the file. (For the TX90L we scaled the puppet to 30% of the target robot).

Click IMPORT

 

Using pocket geometries, modify the robot's mesh to allow the installation of the joint encoder assemblies as well as access for any additional hardware.

In TinkerCAD, copy or import the joint encoder pocket shape into the link file you are working with.

Center the encoder void of the joint encoder pocket shape around the axis of rotation for the joint you are workin on. It is important to ensure that the pocket geometry is properly centered on the axis of rotation or the resulting puppet will not properly mimic the target robot's motion. Using the RULER tool will greatly assist in this process.

Select the pocket geometry and, in the Inspector, set the color of the piece to 'HOLE' (circled in green on the image to the left). This will cause the shape to create a hole in the surface in which it intersects

Select both the link and the pocket geometry and click on 'GROUP (circled in red on the image to the left)'. This will join the two pieces together and create a pocket geometry shaped hole in the joint surface.

Follow the same process for the joint axis pocket.

Using the shapes provided in TinkerCAD add any additional cable management or access points required. In the image to the left you can see the link as a shadowed shape, the joint encoder pocket in pink, the joint axis pocket in orange(with attached frinction bolt access) and a cable management tunnel in green to allow the cables from the joint encoder assembly to be run to the outside of the link.

Eventually you should have a completed, modified mesh that includes all neccesary access tunnels and pocket geometries. It should look something like the model to the left.

Export your mesh for printing by going to DESIGN > DOWNLOAD FOR 3D PRINTING

Select '.STL' and your download will begin shortly.

 

 

 

Print the puppet

Open Makerware Desktop.

Click on 'ADD FILE'

Navigate to downloaded STL file of the modified mesh and click 'OPEN'

Move your obect to the plaftform and rotate it until it is appropriate positioned on the platform

NOTE - In order to minimize support structures and potential printing issues it is benefitial to ensure that the model is placed/oriented on the printing platform in such a way that there is a flat surface on the printing platform and overhangs are minimized as much as possible.

NOTE - It is worthwhile to print any pieces that are completely round or curved in two pieces so that there is at least one flat suface to place on the printer platform. It may be neccesary to add additional access tunnels or mounting holes in order to achieve this goal. An example of this can be seen here, where link 3, the 'upper arm' of the robotic arm was split in two legnth wise and additional mounting holes were added to allow the installion of bolts to keep the two pieces together.

Print the piece using the settings appropriate to your printer. We used the following settings:

Infill - 10%
'Standard Quality' - 2 shells, 0.20 mm height
Rafts - on
Supports - on
Material - ABS

Print the link and monitor for any issues

Print any associated brackets for this link. (each link with a joint assembly pocket should have a matching mounting bracket.

 

 

Clean the printed model and trim any overhand, rafts or support structures.

Trim any excess material from the model including raft and support structures. Drill out any access or wiring tunnels that were inadvertantly filled during the printing process.

With the clean model in hand affix any additional surface adjustments. We installed a thin layer of open-cell foam(craft form) to the joint contact surfaces in order to allow for a smoother action and finer friction control tuning. This foam can be seen in white against the yellow link in the image to the left.

 

 

 

 

 

Install joint controller assemblies.

In the image to the left is an example of an installed joint encoder assembly. The potentiometer shaft should be passed through the central hole in the bracket. Two bolts should be used in the two outside holes in the bracket to attach the assembly to the printed link. Wires should be run through the created wiring tunnels.

 

 

Assemble puppet

Using the joint encoder shafts as connectors, assemble the puppet.

Install friction control bolts

While assembling the puppet install the friction control bolts for each joint

 

 

Run wiring and hook everything up

Wiring should be run following the schematic to the left. The number of articulated joints will determine the board used. The TX90L Puppet has 5 articulated joints, we used the Arduino UNO to read the values from the potentiometers.

Wires should be run through the wiring and access tunnels in each link and enough slack should be given so that the robot's motion is not inhibited.
